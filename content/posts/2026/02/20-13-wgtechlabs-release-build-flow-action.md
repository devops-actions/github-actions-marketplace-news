## Action Summary

The **Release Build Flow Action** automates the creation of GitHub Releases and changelog maintenance by parsing conventional commit messages. It determines semantic version bumps (major, minor, patch), generates structured changelogs using the "Keep a Changelog" format, and publishes releases with zero manual intervention. This action simplifies release management by eliminating the need for manual versioning, changelog updates, and release creation, ensuring consistent and professional workflow automation.
