## Action Summary

The Changelog.Action GitHub Action automates the process of updating the `[Unreleased]` section in a project's `CHANGELOG.md` file to reflect new releases, in accordance with the [Keep a Changelog](https://keepachangelog.com/en/1.0.0/) standard. It also provides a `release-notes` output containing the content of the updated section, which can be used to create GitHub releases or integrate with other automation workflows. This action simplifies release management by ensuring changelogs are consistently maintained and release notes are easily accessible.
