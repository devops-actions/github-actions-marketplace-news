## Action Summary

The `xfg` GitHub Action enables declarative management of files, repository settings, and rulesets across GitHub, Azure DevOps, and GitLab using a single YAML configuration file. It automates tasks such as synchronizing configuration files, applying repository standards (e.g., security settings, branch protection rules), and managing repositories (e.g., creation, forking, migration). This action helps organizations enforce consistent configurations and workflows efficiently across multiple repositories.
