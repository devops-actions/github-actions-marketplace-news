## Action Summary

The **Build Size Diff** GitHub Action automates the tracking and comparison of bundle sizes during CI/CD workflows. It identifies performance regressions by analyzing gzip and brotli-compressed sizes, providing detailed pull request comments with file-level changes, size budgets, and visual diffs. This action is framework-agnostic, works with any bundler, and helps developers enforce size constraints, ensuring optimized builds before merging code.
