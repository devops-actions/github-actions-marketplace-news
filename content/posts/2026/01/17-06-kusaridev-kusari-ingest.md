## Action Summary

The `kusari-ingest` GitHub Action automates the ingestion of artifacts, such as SBOMs, SLSA, and other attestations, into the Kusari Platform as part of a CI/CD workflow. It simplifies the integration process by leveraging authentication credentials provided by Kusari and enables users to streamline artifact management and security compliance checks. Key features include support for various artifact types, optional validation against blocked package lists, and customizable settings for ingestion and reporting.
