## Action Summary

The `kusari-ingest` GitHub Action automates the ingestion of various software artifacts, including SBOMs, SLSA, and other attestations, into the Kusari Platform as part of a CI/CD workflow. This action simplifies integration with the Kusari Platform, enabling organizations to efficiently manage software supply chain data and perform security checks, such as verifying dependencies against a blocked package list. It provides seamless authentication and flexible options for organizing and tagging ingested documents.
