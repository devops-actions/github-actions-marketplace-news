## Action Summary

The **repo-task-tracker** GitHub Action automates the creation and synchronization of GitHub Issues and their corresponding entries in a centralized GitHub Projects (v2) board based on tasks defined in a `tracker.json` file within your repository. It streamlines task management across multiple repositories by automatically updating issue statuses, labels, and project board fields (e.g., status, priority) using GitHub's GraphQL API, enabling users to efficiently track and manage tasks from a single location. This action is particularly useful for maintaining consistent project tracking across multiple repositories or collaborative projects.
