## Action Summary

The `repo-sunsetter` GitHub Action automates the process of preparing a repository for archival by adding an archival notice to the README, updating project metadata, and creating an issue with a tailored archival review checklist. It simplifies repository sunsetting by ensuring that necessary documentation and metadata updates are completed systematically, while offering customization options for different project types and requirements. This action is particularly useful for maintaining transparency and consistency when archiving repositories, based on maturity models or custom criteria.
