## Action Summary

The **github-screenshot-action** automates the process of capturing, monitoring, and versioning website screenshots. It uses a JSON-driven configuration to define target URLs, supports parallel execution, retry logic, and scheduling via cron jobs, and can automatically create pull requests with updated screenshots. This action is ideal for monitoring visual changes on websites over time, offering capabilities like real-browser rendering with Puppeteer, flexible wait strategies, and isolated branch management for screenshot updates.
