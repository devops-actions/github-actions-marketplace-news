## Action Summary

The `semVersie` GitHub Action automates the calculation of semantic version bumps (major, minor, patch) based on conventional commit messages in pull requests. It helps streamline the versioning process by determining the next release version and optionally labeling pull requests with their version impact. This action simplifies release management without restricting usage to specific programming languages, making it a versatile tool for projects following semantic versioning practices.
