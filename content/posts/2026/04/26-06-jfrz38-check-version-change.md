## Action Summary

The `check-version-change` GitHub Action automates the process of comparing a project's current version (from a specified file) against a version in a package registry, marketplace, or Git reference. It identifies version changes, checks if the current version is semantically higher, and provides workflow outputs to conditionally trigger actions like publishing. This action helps enforce version control policies, gate publishing workflows, and ensure consistency across versioned artifacts.
