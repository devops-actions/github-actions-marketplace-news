## Action Summary

The `csda-version` GitHub Action automates the calculation of the next CSDA version for a repository using a structured versioning scheme (`vYY.PI.SP-X`) based on year, program increment (quarter), sprint, and release number. It integrates seamlessly with tools like `release-please` to streamline release management by dynamically determining and applying version numbers. This action simplifies versioning consistency, automates routine updates, and ensures changelog generation includes relevant commits based on predefined prefixes.
