## Action Summary

The `build-changelog` GitHub Action automates the process of generating and managing changelogs by allowing contributors to add change details in individual files, which are then compiled into a structured changelog. Inspired by "Keep a Changelog" and "Common Changelog," it supports semantic versioning and predefined change categories (e.g., added, changed, fixed). This action simplifies changelog maintenance, enforces consistent formatting, and integrates seamlessly into CI workflows to streamline release management.
