## Action Summary

The **GitHub Omni Comment Action** consolidates outputs from multiple CI jobs into a single, organized comment on a pull request, with customizable sections defined in a configuration file. It solves the problem of cluttered PR discussions caused by multiple individual comments, streamlining feedback by grouping information like test results, deploy previews, and performance stats into one unified comment. Key capabilities include support for dynamic updates, section ordering, optional titles, collapsible sections, and commenting on specific pull requests.
