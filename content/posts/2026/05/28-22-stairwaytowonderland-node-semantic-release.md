## Action Summary

The `node-semantic-release` GitHub Action automates the process of building a Node.js project, releasing it using `semantic-release`, and optionally creating or managing GitHub Releases with encoded release notes for downstream workflows. It supports two modes: a default "Release mode" that handles the entire release pipeline, including tagging and asset creation, and a "Publish mode" for creating a GitHub Release from an existing tag. This action simplifies release management by streamlining dependency installation, versioning, and release note generation while addressing common challenges like triggering downstream workflows.
