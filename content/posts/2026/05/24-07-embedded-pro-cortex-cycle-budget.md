## Action Summary

The `cortex-cycle-budget` GitHub Action provides a static analysis tool for estimating the cycle cost of critical paths in ARM Cortex-M firmware, specifically for real-time systems like motor control and signal processing. It automates the analysis of ELF binaries to ensure that control loops fit within predefined cycle budgets, offering features such as annotated disassembly, CPU utilization reports, machine-readable metrics, and CI integration for automated performance gating. The action supports multiple Cortex-M variants and includes timing models, FPU costs, and exception overhead modeling, making it a versatile solution for firmware optimization and validation.
