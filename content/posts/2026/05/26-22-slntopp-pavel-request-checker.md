## Action Summary

The `pr-title-check` GitHub Action automates the validation of pull request titles against predefined structural rules, ensuring consistency and clarity in naming conventions. It helps teams enforce standards such as required emoji prefixes, word count, character limits, and banned phrases, with optional AI-powered semantic checks for title relevance. This action simplifies title management by providing default rules, customizable configurations, and actionable feedback via sticky PR comments.
