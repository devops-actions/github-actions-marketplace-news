## Action Summary

The **`node-semantic-release`** GitHub Action automates the process of releasing Node.js projects by installing dependencies, building the project, running `semantic-release` to determine versioning and generate release notes, and optionally creating GitHub releases with tags and assets. It simplifies release management by supporting two modes: a default **Release mode** for end-to-end automation including tag creation, and a **Publish mode** for creating GitHub releases from existing tags. This action streamlines semantic versioning, release note generation, and release asset handling to enhance CI/CD workflows.
