## Action Summary

The `should-run` GitHub Action evaluates whether a downstream job should execute based on file changes matching specified include/ignore patterns. It addresses the issue of workflows being stuck in "pending" for documentation-only changes when used as a required status check in branch protection. By automating this decision-making process and exposing matched file lists, it ensures workflows properly report statuses and supports finer-grained control in subsequent steps.
