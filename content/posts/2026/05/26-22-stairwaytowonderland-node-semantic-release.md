## Action Summary

The `node-semantic-release` GitHub Action automates the process of building Node.js projects, generating semantic releases, and managing GitHub Releases. It supports two modes: a default "release mode" to build the project, create semantic versions, generate release notes, and optionally tag releases, and a "publish mode" to create GitHub Releases from pre-existing tags and release notes. This action streamlines release workflows by handling versioning, changelog generation, and release publication, while also offering flexibility for different token and automation configurations.
