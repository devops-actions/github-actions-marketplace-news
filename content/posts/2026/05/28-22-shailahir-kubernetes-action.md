## Action Summary

This GitHub Action streamlines Kubernetes deployments by automating tasks such as applying manifests, managing multiple deployment files, triggering rollout restarts, monitoring rollout health, and verifying deployment success. It ensures robust CI/CD pipelines by failing workflows on unhealthy rollouts and providing detailed insights into pod statuses and deployment health. Key features include namespace-aware deployments, dry-run support, and user-friendly logging tailored for GitHub Actions.
