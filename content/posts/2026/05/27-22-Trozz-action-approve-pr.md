## Action Summary

This GitHub Action, **approve-pr**, facilitates human approval workflows for pull request jobs by requiring designated approvers to react to a comment posted on the pull request. It automates the gating process by approving, rejecting, or failing a job based on specific reactions (e.g., `+1` or `-1`) from authorized users or teams, and ensures approvals are tied to the exact commit to prevent carryovers after code changes. Key features include support for team-based approvals, customizable reactions, flexible timeout settings, and detailed approval tracking.
