## Action Summary

**PinnedAI** is a GitHub Action and tool that creates a repository-specific safety net for AI-assisted coding by establishing baseline security guards, such as protecting authentication checks, lockfile integrity, and sensitive code patterns. It automates the prevention of unauthorized changes or bypass attempts, generates AI-readable development guidelines, and ensures AI-generated code complies with defined safeguards. The action integrates seamlessly with popular AI coding tools, CI pipelines, and pre-commit workflows to enhance code security and maintain project integrity.

## What it does

1. Scans your repo for risk surfaces (auth, webhooks, routes, env files, lockfile drift) and generates baseline regression guards as Vitest tests.
2. Installs a pre-commit hook that blocks `.skip()` / `.only()` / weakened-assertion / deleted-test bypass attempts (17/17 known AI bypass tactics blocked in the mutation-test suite).
3. Writes AI-coder rules to `CLAUDE.md` and `.github/copilot-instructions.md` so Claude / Cursor / Copilot read them before editing.
4. Adds `.github/workflows/pinned.yml` so CI also enforces guard integrity (catches `git commit --no-verify` bypass attempts).
